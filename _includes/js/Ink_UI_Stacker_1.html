Ink.UI.Stacker_1

METHODS
Method name Description
.addItem

Add an item to the end of your stacks. Call reloadItems() when you are done adding items.
.reloadItems

Update the layout of your items.

Call this when:

  • The width has changed, but not because of the window resizing.
  • You used addItem to add some items

This module combines several stacks of items together, in smaller screen sizes.

The purpose is to have several stacks of items which may have different heights and as such cannot be used because of float: left quirks.

For example, when you have three different columns of information:

[col. A: 1] [col. B: 1] [col. C: 1]
[col. B: 2] [col. C: 2] [col. C: 2]

and the screen resizes and you need a layout of 2 columns, Stacker reorders the stacks so that you get:

[col. A: 1] [col. B: 1]
[col. C: 1] [col. A: 2]
[col. B: 2] [col. C: 2]

Note: If you just want to use a different amount of columns for your items in several viewports, but these items are guaranteed to have a fixed height, don't use this module. Use the small-*, medium-* and large-* classes instead.

Ink.UI.Stacker_1 ([container], [options], ) constructor

  • {DOMElement|String} Element which contains the stacks (identified by the options.column selector)
  • {String} Select the columns inside the `container`
  • {String} Select items in your stack
  • {Object} options for each breakpoint name. Use this if you have more breakpoints than Ink by default (`large`, `medium`, `small`)
  • {Object}
  • Maximum screen size as seen in your media query
  • Minimum screen size as seen in your media query
  • Column count for this size.
  • {Number} Upper bound of `large` breakpoint
  • {Number} Lower bound of `large` breakpoint
  • {Number} Upper bound of `medium` breakpoint
  • {Number} Lower bound of `medium` breakpoint
  • {Number} Upper bound of `small` breakpoint
  • {Number} Lower bound of `small` breakpoint
  • {Integer} Number of columns in the `large` viewport
  • {Integer} Number of columns in the `medium` viewport
  • {Integer} Number of columns in the `small` viewport
  • {Boolean} When `false`, do not reorder stacks when combining them.
  • {Function} Called when instantiated
  • {Function} Called when resized because the window resized.
  • {Function} Called when you ask to reload stack items from the DOM.
