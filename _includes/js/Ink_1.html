Ink.createModule (mod, version, deps, modFn) method 

  • String
    Module name, separated by dots. Like Ink.Dom.Selector, Ink.UI.Modal 
  • Number
    Version number 
  • Array
    Array of module names which are dependencies of the module being created. The order in which they are passed here will define the order they will be passed to the callback function. 
  • Function
    The callback function to be executed when all the dependencies are resolved. The dependencies are passed as arguments, in the same order they were declared. The function itself should return the module.

Creates a new Ink Module. Use this to wrap your code and benefit from the module loading used throughout the Ink library

Ink.setPath (key, rootURI) method 

  • String
    Module or namespace 
  • String
    Base URL path and schema to be appended to the module or namespace

Sets the URL path for a namespace or module. Use this to customize where requireModules (and createModule) will load dependencies from. This can be useful to set your own CDN for dynamic module loading or simply to avoi

Example

{% highlight html %}{% raw %} Ink.setPath('Ink', 'http://my-cdn/Ink/'); Ink.setPath('Lol', 'http://my-cdn/Lol/'); // Loads from http://my-cdn/Ink/Dom/Whatever/lib.js Ink.requireModules(['Ink.Dom.Whatever'], function () { ... }); // Loads from http://my-cdn/Lol/Whatever/lib.js Ink.requireModules(['Lol.Whatever'], function () { ... }); {% endraw %}{% endhighlight %}
