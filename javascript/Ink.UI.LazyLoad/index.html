Ink.UI.LazyLoad_1version:1


This delays the loading of images and other content until the corresponding elements are visible in the browser viewport.
This was created to load images later, but can be also used for widgets which are slow to load and are only useful when on screen.


All samples are using Ink.requireModules, please read how to use it at Ink.requireModules section

Accepts

{String|DOMElement} The element which contains the lazily-loaded items.
ObjectOptions object, containing:
StringItem selector. Defaults to '.lazyload-item'.
StringPlaceholder value for items which are not 'visible', in case they don't already have a value set.
StringSource attribute. When an item is 'visible', use this attribute's value to set its destination attribute. Defaults to 'data-src'.
StringDestination attribute. Attribute to change when the element is 'visible'. Defaults to 'src'.
NumberMilliseconds to wait before trying to load items. Defaults to 100.
NumberOffset distance in pixels. Determines how far the top of an item must be from the viewport be considered 'visible'. Negative values shrink the considered 'visible' viewport while positive values enlarge it. Defaults to 0.
BooleanSet to false to make this component do nothing to any elements and just give you the onInsideViewport callback.
DOMElement(advanced) What element is to be listened for the scroll event. Defaults to document.window.
BooleanSubscribe to touch events in addition to scroll events. Useful in mobile safari because 'scroll' events aren't frequent enough. Defaults to true.
FunctionCallback function for when an `item` is 'visible'. Receives an object containing the item's element as an argument.
Function(advanced) Callback function when an item's attribute changes. Receives an object containing the item's element as an argument.
Boolean(advanced) Set to false if you want to start LazyLoad yourself with `reload()`. Defaults to true.

.reload() method

Load or reload the component. Adding the 'scroll' event listener if necessary and checks if anything needs to be loaded now.

You can use this to manually invoke the loading logic without user action.
