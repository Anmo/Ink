Introduction

Our JavaScript library has a modular architecture and our code is organized in namespaces:

  • Ink - Modulo base onde existem métodos transversáis a todos os módulos
  • Ink.Dom - Módulos que tenham qualquer tipo de interação com o DOM (criar e manipular elementos, gestão de eventos e serialização de formulários)
  • Ink.Net - Módulos de comunicação AJAX e JsonP
  • Ink.UI - Módulos com métodos utilitários.
  • Ink.Util - Módulos com métodos utilitários.
  • Ink.Ext - Módulos de extensões, que não se enquadram em nenhum dos outros namespaces ou sem maturidade suficiente para fazerem parte dos namespaces core.

The Core provides the foundation for all other components, such as DOM manipulation, event management, AJAX and JsonP requests along with tons of additional utilities.

The UI module contains visual components that allow you to enhance your pages with interactivity, like carousels, modal windows, tooltips, etc.

The Ink.Ext is a special namespace. It was created to allow the community to contribute with new modules or components. These contributions may eventually be integrated with Inks core. Read more on how to contribute on our Github repository.

Quando o site carrega o autoload.js é executado, irá procurar os módulos identificados no site, para cada um recolhe as opções definidas com data-attributes e instancia o módulo passando as opções recolhidas. - Integração instanciando com JavaScript - Usando o Ink.requireModules define-se o módulo pertendido com a respectiva versão e instancia-se passando o selector e as opções. Core: A biblioteca está organizada nos seguintes namespaces: Ink - Modulo base onde existem métodos transversáis a todos os módulos Ink.Dom - Módulos que tenham qualquer tipo de interação com o DOM (criar e manipular elementos, gestão de eventos e serialização de formulários) Ink.Net - Módulos de comunicação AJAX e JsonP Ink.Util - Módulos com métodos utilitários. Ink.Ext - Módulos de extensões, que não se enquadram em nenhum dos outros namespaces ou sem maturidade suficiente para fazerem parte dos namespaces core. Como usar o Ink core no seu projecto? Recomendamos que use o Ink.requireModules() indicando os nomes dos módulos e escreva o código dentro da função de callback. Desta forma conseguirá gerire melhor quais as dependências e não irá poluir o scope global com variáveis. Todos os exemplos da documentação estão o usar o Ink.requireModules... não tem necessidade de o utilizar para a invocação de cada método. Veja o exemplo em baixo de como seria o ficheiro de JavaScript do seu projecto. Ex: /* Start of my_project.js file */ Ink.requireModules([ 'Ink.Dom.Loaded_1', 'Ink.Dom.Element_1', 'Ink.Dom.Event_1', 'Ink.Dom.Css_1', 'Ink.Net.Ajax_1', 'Ink.UI.Modal_1', 'Ink.UI.Carousel_1' ], function(Loaded, InkElement, InkEvent, InkCss, Ajax, Modal, Carousel){ var body = Ink.s('body'); var container = Ink.ss('#container')[0]; function onClickBody() { InkCss.addClassName(body, 'body-class'); var newElm = InkElement.create('div', { className:'some-class', id: 'newdiv', setTextContent: 'Element text content', insertBottom: container }); } function addEvents() { InkEvent.on(body, 'click touchstart', onClickBody); } /* ... more code ... */ function initProject() { addEvents(); new Modal('#project-modal', {trigger: '#open-modal'}); new Carousel('#project-carousel', {pagination: '#carousel-pagination'}) } Loaded.run(function() { // will run on DOMContentLoaded initProject(); }); }); /* End of my_project.js file */ Leia com atenção a página de documentação do namespace Ink, em especial os métodos Ink.requireModules, Ink.createModule, Ink.bind, Ink.bindEvent, Ink.i, Ink.s e Ink.ss. Se pretender criar os seus próprios módulos de Ink, recomendamos que utilize o Ink.createExt(). O módulo ficará disponível no namespace Ink.Ext. Se disponibilizar a sua extensão num repositório público e nos informar, teremos todo o prazer em o listar na nossa página de extensões Ink (a página não está criada ou então usar a de projectos que usam Ink que já existe)
