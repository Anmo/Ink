Ink.Dom.FormSerialize_1

Two way serialization of form data and javascript objects. Valid applications are ad hoc AJAX/syndicated submission of forms, restoring form values from server side state, etc.

All samples are using Ink.requireModules, please refer to stuff

Functions
Function name Description
.fillIn

Sets form elements' values with values from an object

Note: You can't set the values of an input with type="file" (browser prohibits it)
.serialize

Serializes a form element into a JS object It turns field names into keys and field values into values.

note: Multi-select and checkboxes with multiple values will result in arrays
