Ink.UI.TreeView_1version:1

Shows elements in a tree structure which can be expanded and contracted.
A TreeView is built with "node"s and "children". "node"s are li tags, and "children" are ul tags.
You can build your TreeView out of a regular UL and LI element structure which you already use to display lists with several levels.
If you want a node to be open when the TreeView is built, just add the data-open="true" attribute to it.

All samples are using Ink.requireModules, please read how to use it at Ink.requireModules section

Methods
Method name Description
(constructor) Constructor
.close Closes one of the tree nodes

Ink.UI.TreeView(selector, [options], ) constructor

  • Arguments:
  • selector
  • [options]

Accepts

String|DOMElement Element or selector.
String Options object, containing:
String Selector for the nodes. Defaults to 'li'.
String Selector for the children. Defaults to 'ul'.
String CSS classes to be added to parent nodes. Defaults to 'parent'.
String CSS classes to be added to the icon when a parent is open. Defaults to 'fa fa-minus-circle'.
String CSS classes to be added to the icon when a parent is closed. Defaults to 'fa fa-plus-circle'.
String CSS Class to toggle visibility of the children. Defaults to 'hide-all'.
String The name of icon tag. The component tries to find a tag with that name as a direct child of the node. If it doesn't find it, it creates it. Defaults to 'i'.
Boolean Flag to stops the default behavior of the click handler. Defaults to true.
