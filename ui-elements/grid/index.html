Introduction

Our layout grid was built to allow precise control on how horizontal space is divided and on targeting several screen sizes. To accomplish this we use percentages as our base units and css @media queries to allow you to decide what width should an element take when viewed in a specific screen size.

Our default layout breakpoints are:

  • tiny for screens widths up to 320px.
  • small for screen widths between 321 and 600 pixels.
  • medium for screen widths between 601 and 960 pixels.
  • large for screen widths between 961 and 1200 pixels.
  • extra-large for screen widths above 1201 pixels.

In many situations in Ink you'll find that you can use the breakpoint name as a css class prefix to apply a certain behavior based on screen size, e.g. .large-align-right will cause text to be aligned to the right when the screen width is between 961 and 1200 pixels.

To use the grid you'll need a block level element, usually a div, with the .ink-grid class.

Dividing space

We have 22 width units, that we'll call columns, ranging from 5% to 100% in 5% increments as well as 33% and 66% for one and two thirds respectively.

Additionally we have a grid class that is used to group columns so they always stay together and don't get tangled with other layout elements. All these grid units are available as css classes that are prefixed with the breakpoints names:
