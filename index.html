More than HTML and CSS

We've developed our own Javascript library based on the latest coding standards and a modular architecture. Using Ink.requireModules() method make sure all of Ink power is at your finger tips.

{% highlight js %} Ink.requireModules( ['Ink.Dom.CSS_1', 'Ink.Dom.Selector_1'], function(c, s) { var myDiv = Ink.s('#intro'); c.addClass(myDiv,'hot'); } ); {% endhighlight %}

Customizable

Our CSS is created with SASS. Dimensions, color, font sizes and even class names are defined as variables and are easily reconfigurable. Just edit a couple of files, recompile and you'll have your unique build.

Why?

I'm new to UI frameworks, why would i use one?

  • Get things done faster by reusing code and a familiar syntax.
  • Using a solid code base makes your work open to others.
  • Eliminate continuous code rewriting/refactoring.

I'm familiar with UI frameworks, why would i use Ink?

  • Everything is fluid and everything is responsive.
  • Limitless options for layout creation with our grid.
  • Absolute control over almost all aspects of your layout and how it is displayed on different screen sizes.
  • Human readable syntax.
  • Built for designers and developers alike.

How?

Generated

Inks CSS is created with SASS and uses Compass to make our work a lot easier. This also makes it very easy for you to do the same.

Customizable

Allmost all aspects of Ink, like dimensions, coloring, typography and even class names are defined with SASS variables. Creating your own version is as simple as editing a few files and recompiling.

Modular

If you need just a couple of our components, just edit the code and recompile your stripped down version of our CSS.

Javascript

Ink.js is our own library and comes packed with lots of UI components, and the tools to let you build your own.
